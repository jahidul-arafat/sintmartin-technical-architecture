About This Platform

What is this? This is a national e-services interoperability platform — a shared digital infrastructure that lets citizens and businesses interact with government services without re-entering information the government already holds. When a citizen applies for a building permit, business licence, or social benefit, the platform automatically prefills their form with verified data from six national registries (Civil, Address, Business, License, Tax, Land). They simply review, sign, pay if required, and submit. Behind the scenes, the platform orchestrates every data fetch, consent check, document scan, eSign, and payment in a secure, auditable, and GDPR-compliant way.

Why does this matter? Government services have traditionally been siloed: each ministry maintains its own citizen portal, its own authentication, and demands the same personal data repeatedly. This platform breaks that pattern by acting as a single, consent-governed data broker. A citizen registers once with their national eID; thereafter every participating service can, with explicit consent, reuse verified data from authoritative registries. This is the EU Once-Only Principle — a legal obligation for digital government under the Single Digital Gateway Regulation. The platform makes this technically concrete and auditably compliant.

Design strategy. The architecture is deliberately layered and loosely coupled. At the edge, Azure Front Door and APIM Premium form a zero-trust gateway — every request is authenticated, RBAC-checked, and rate-limited before it reaches any business logic. Behind the gateway, an API Facade provides versioned, stable contracts so consumer applications never break when internal implementation changes. The Orchestrator coordinates multi-step workflows (consent → prefill → eligibility → document → eSign → payment → submit → case) using a saga pattern with idempotency and compensating transactions. Six registry adapters connect to authoritative sources through a canonical DTO model so data quality rules run once, not per-registry. All operational data writes go to Azure SQL MI (zone-redundant, PITR-enabled) with a parallel event-driven lakehouse pipeline feeding Bronze → Silver → Gold Delta Lake tables in Fabric for analytics and reporting. The platform is deployed across two Azure regions: Region A as the zone-redundant primary, Region B as a warm DR standby with RPO ≤ 5 minutes and RTO ≤ 60 minutes — validated by quarterly failover drills.

Design Tenets
🔒
Zero-Trust Security
All traffic enters through Front Door WAF → APIM JWT validation → Facade ABAC. Short-lived tokens (≤15 min), ACR step-up for high-assurance operations. No implicit network trust — every request must be authenticated and authorised at every layer.
✋
Consent-First & Privacy-by-Design
Consent gates enforced before every registry call. Purpose-bound, scoped, time-limited consents stored in the Consent Service. Immutable WORM audit receipts (JWS-signed) for every key operation. GDPR-native data minimisation and expiry.
♻
Once-Only Principle
Citizens never re-enter data the government already holds. Cache-first MDM prefill with ETag/TTL. Event-driven cache invalidation from SoR. SLO: hit-rate ≥80%, stale field rate <2%, consent-gate violations = 0. P95 ≤600ms.
🔗
API-First, No Point-to-Point
All registry access flows through APIM → Facade → Orchestrator → Adapters. Versioned stable contracts (/v1…/v2) with Deprecation/Sunset headers. No service bypasses the API Gateway. Canonical DTO normalisation across all 6 adapters.
📡
Resilient & Observable
Retries + circuit breakers + DLQ for all external calls. SLO dashboards, KQL burn-rate alerts. RPO ≤5 min / RTO ≤60 min. Zone-redundant primary region (Region A) + warm DR standby (Region B). Pre-warmed, runbook-guided failover.
